# Funwall planning repository

This repository currently contains the complete clean-room build specification for a personal Wordwall-like activity website. No application code has been written yet.

Start here:

1. [`FUNWALL_MASTER_PLAN.md`](FUNWALL_MASTER_PLAN.md) — product, research, architecture, six game specifications, milestones, and final definition of done.
2. [`agent-work/README.md`](agent-work/README.md) — five-agent, three-agent, maximum-parallel, and solo execution workflows.
3. [`agent-work/shared/CONTRACTS.md`](agent-work/shared/CONTRACTS.md) — shared boundaries that keep independently built activities compatible.
4. Choose one compartmentalized [`agent-work/*/TASK.md`](agent-work/README.md#2-workstream-index) packet.
5. End every implementation packet with [`agent-work/shared/HANDOFF_TEMPLATE.md`](agent-work/shared/HANDOFF_TEMPLATE.md).

Recommended first build action: assign the Integration Lead packet and Foundation/Data packet, freeze code-level contracts, then finish the deployed saved Spin the Wheel vertical slice before starting all other games in parallel.
